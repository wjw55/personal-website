# AGENTS.md

## Project goal

This repository contains a personal website for [YOUR NAME]. The site should be polished, fast, accessible, responsive, and easy to maintain.

The website should present:
- A clear personal introduction
- Selected projects
- Professional background
- Writing or notes
- Contact and social links

Use placeholder content when personal details are missing. Never invent specific biographical facts, employers, degrees, awards, or project claims. Mark placeholder content clearly so it can be replaced later.

## Preferred stack

Use:
- Next.js App Router
- TypeScript
- Tailwind CSS
- Static-first pages and components
- Simple local content files where possible

Avoid:
- Unnecessary backend services
- Databases unless explicitly requested
- Authentication unless explicitly requested
- Paid APIs or third-party services unless explicitly approved
- Storing secrets in the repo

## Design direction

Aim for:
- Clean, modern, minimal layout
- Strong typography
- Generous spacing
- Responsive design
- Accessible color contrast
- Professional but personal tone

The site should work well on:
- Mobile
- Tablet
- Desktop

## Content rules

Use these placeholders until real information is provided:
- Name: [YOUR NAME]
- Headline: [YOUR ONE-LINE HEADLINE]
- Bio: [SHORT BIO]
- Email: hello@example.com
- GitHub: https://github.com/yourusername
- LinkedIn: https://linkedin.com/in/yourusername

Do not include:
- Private phone numbers
- Home address
- Date of birth
- Government IDs
- Passwords
- API keys
- Private employment details
- Sensitive personal information

## Code quality

When changing code:
- Prefer small, readable components
- Keep file names clear and consistent
- Use semantic HTML
- Ensure keyboard accessibility
- Add alt text where images are used
- Avoid over-engineering
- Prefer simple data structures for editable content
- Keep styling consistent

Before finishing a task:
- Run linting if available
- Run type checks if available
- Run formatting if available
- Check for obvious responsive layout issues
- Update the README if setup or usage changes

## Definition of done

A task is complete when:
- The site builds successfully
- The main pages are responsive
- Placeholder content is easy to find and replace
- There are no obvious accessibility issues
- The README explains how to run and customize the site
- The final response summarizes changes and any remaining TODOs
